USB Rubber Ducky Keylogger Payload

A payload development project focused on offensive scripting, keystroke injection, and the collection of system and network information for security research and demonstration.

Project Overview

The objective of this project is to simulate a USB-based HID attack using a Rubber Ducky device to demonstrate how physical access can lead to system information disclosure. The payload is designed to raise awareness around endpoint vulnerabilities by automatically collecting system details (e.g., IP addresses, hostname, Wi-Fi SSID) and exfiltrating them to a remote GitHub repository. This proof-of-concept highlights the importance of USB security, physical device control, and endpoint monitoring in an organization's defense strategy.

Methodology

Phase Activities Tools & Techniques
Initial Access Physical insertion of USB Rubber Ducky device HID emulation, DuckyScript injection
Execution Automated command execution and system information gathering PowerShell scripts, system enumeration commands
Collection Gathering sensitive system data (IP, hostname, Wi-Fi credentials) Network configuration queries, credential harvesting
Exfiltration Secure transmission of collected data to remote repository GitHub API, HTTPS encryption, RESTful communication
Anti-Forensics Masking payload activity and cleanup Visual distractions, temporary file management

Key Findings

Testing conducted in controlled environment with explicit authorization. Payload designed for educational purposes only.

Recommendations

  • Implement USB port security controls and device whitelisting
  • Configure PowerShell execution policies and script block logging
  • Deploy endpoint detection and response (EDR) solutions
  • Enable network traffic monitoring for unauthorized connections
  • Conduct regular security awareness training on physical threats
  • Implement physical access controls and device lockdown policies
  • Monitor for anomalous API calls and data exfiltration attempts
  • Apply principle of least privilege to user accounts and systems

Technical Implementation

The payload was written in DuckyScript and leverages PowerShell and Bash scripts for execution. It demonstrates advanced techniques in offensive security tooling, including persistence mechanisms and anti-forensic measures.

Conclusion

This Rubber Ducky payload project demonstrates how a simple HID-based attack can be used to exfiltrate sensitive system information within seconds of physical access. While the payload itself is non-malicious and intended for awareness, it highlights how easily a system can be compromised without the need for elevated privileges or complex malware. The experiment underscores the importance of physical security, USB port management, user training, and endpoint monitoring as critical layers of defense.
